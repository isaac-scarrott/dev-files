# dev-files

My dotfiles. The repo is the source of truth on disk: anything tracked here is symlinked into its normal place, so editing a config in its usual location is editing this repo. `git status` will show it.

## New machine

```bash
git clone --recurse-submodules https://github.com/isaac-scarrott/dev-files ~/git/dev-files
cd ~/git/dev-files
./install.sh
```

`install.sh` is idempotent and safe to re-run. It points `core.hooksPath` at `.githooks`, inits submodules, runs `scripts/vendor.sh` to fetch upstream files, then walks the `LINKS` array and symlinks each entry into your home dir. Existing real files get backed up to `*.bak.<timestamp>` before anything is replaced.

macOS only right now (Library paths are baked in).

## Layout

```
.claude/         claude code: skills, scripts, hooks, settings
.config/         editor + wm config (nvim, zed, aerospace, ghostty, alacritty)
.githooks/       pre-commit re-runs vendor.sh when the manifest changes
ide/             vscode + cursor settings.json (live under Library/Application Support/)
scripts/         vendor.sh, drift.sh, test-install.sh
.zshrc           shell
.tmux.conf       tmux
install.sh       bootstrap (and source of the LINKS array)
vendor.manifest  upstream files to keep in sync
```

A couple of things live here for the symlink but aren't tracked: `.claude/CLAUDE.md` (personal, gitignored) and `.config/nvim/lazy-lock.json` (regenerated by `:Lazy update`, noisy).

## Vendoring upstream files

`vendor.manifest` pulls individual files from public repos without registering each as a submodule:

```
# format: OWNER/REPO REF SRC_PATH DEST_PATH
mattpocock/skills main skills/productivity/grill-me/SKILL.md .claude/skills/grill-me/SKILL.md
```

Add a line, stage `vendor.manifest`, commit. The pre-commit hook re-runs `scripts/vendor.sh` and stages the fetched file for you. Use a commit SHA instead of `main` when you want it pinned.

`humanizer` is a real submodule because it's a whole repo, not a file or two.

## Drift

`scripts/drift.sh` checks three things against the declared state:

1. every entry in `install.sh`'s `LINKS` resolves to the right path in this repo
2. every vendored file matches its upstream byte-for-byte
3. each submodule's pinned SHA matches upstream HEAD on its tracking branch

```bash
./scripts/drift.sh         # read-only report
./scripts/drift.sh --fix   # re-vendor, bump submodules, re-link
```

`--fix` refuses to run on a dirty tree, so anything it changes lands in a reviewable diff.

## Tests

```bash
./scripts/test-install.sh
```

Sandboxed in a `mktemp -d` HOME. Covers fresh install, idempotent re-run, backup of pre-existing real files, and replacement of wrong-target symlinks. Runs in about a second, no Docker.

## Gotchas

- Editors that "save by atomic replace" (write new file, rename over the old) will turn a symlink into a real file and silently end the source-of-truth chain. Neovim, Zed, VS Code and Cursor all save in place by default and don't trip this, but worth knowing if a config ever stops syncing.
- `core.hooksPath` doesn't travel with the repo (security thing). `install.sh` sets it on every clone. If you ever skip the install step, the manifest auto-refetch won't fire.
- `skill-creator/LICENSE.txt` will look like drift forever: local has the copyright line filled in, upstream still ships the Apache template placeholder. Local is right.
